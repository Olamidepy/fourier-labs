# Security Policy

## Supported Versions

We actively maintain and support the following versions of Fourier with security patches:

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < v0.1  | :x:                |

---

## Reporting a Vulnerability

**DO NOT open a public GitHub issue for security vulnerabilities.**

If you discover a vulnerability within this codebase (e.g. data leak, validation bypass, injection vector, denial of service), please report it to us privately through the following channels:

- **Security Email:** [security@fouriercore.org](mailto:security@fouriercore.org)

Please include the following details in your report to help us evaluate and address the issue efficiently:
- A clear description of the vulnerability.
- Steps or a proof-of-concept (PoC) script demonstrating the exploit.
- Potential impact on users or infrastructure.

We will acknowledge receipt of your report within 48 hours and work to resolve the issue as quickly as possible. We practice responsible disclosure and appreciate your collaboration in keeping the ecosystem safe.
